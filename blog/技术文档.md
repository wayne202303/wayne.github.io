## 系统架构

### 整体架构
博客管理系统采用前后端分离架构：

```
┌─────────────────┐
│   前端界面      │
│  (HTML/CSS/JS) │
└────────┬────────┘
         │ HTTP/REST API
         │
┌────────▼────────┐
│   后端服务器    │
│   (Python)      │
└────────┬────────┘
         │
┌────────▼────────┐
│  数据存储       │
│  (JSON文件)     │
└─────────────────┘
```

### 模块划分

#### 前端模块
1. **index.html** - 博客首页
2. **admin.html** - 管理后台
3. **css/style.css** - 博客样式
4. **css/admin.css** - 管理后台样式
5. **js/app.js** - 博客前端逻辑
6. **js/admin.js** - 管理后台逻辑

#### 后端模块
1. **server.py** - HTTP服务器
2. **data/posts.json** - 文章数据存储

## 数据库设计

### 数据结构 - posts.json

```json
[
  {
    "id": 1,
    "title": "文章标题",
    "slug": "article-slug",
    "date": "2024-04-14T10:00:00.000Z",
    "category": "分类名称",
    "tags": ["标签1", "标签2"],
    "excerpt": "文章摘要",
    "content": "# Markdown内容",
    "updated_at": "2024-04-14T11:00:00.000Z"
  }
]
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Integer | 是 | 文章唯一标识 |
| title | String | 是 | 文章标题 |
| slug | String | 是 | URL友好的标识 |
| date | String | 是 | 发布时间 (ISO 8601) |
| category | String | 否 | 文章分类 |
| tags | Array | 否 | 标签列表 |
| excerpt | String | 否 | 文章摘要 |
| content | String | 是 | 文章内容 (Markdown) |
| updated_at | String | 否 | 更新时间 |

## API接口文档

### 基础信息
- **Base URL**: `http://localhost:8000`
- **数据格式**: JSON
- **字符编码**: UTF-8

### 接口列表

#### 1. 获取文章列表
```
GET /data/posts.json
```

**响应示例**:
```json
[
  {
    "id": 1,
    "title": "文章标题",
    "slug": "article-slug",
    "date": "2024-04-14T10:00:00.000Z",
    "category": "分类名称",
    "tags": ["标签1", "标签2"],
    "excerpt": "文章摘要",
    "content": "# Markdown内容"
  }
]
```

#### 2. 保存文章数据
```
POST /api/save
Content-Type: application/json

{
  "posts": [文章数据数组]
}
```

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| posts | Array | 是 | 完整的文章数组 |

**响应示例**:
```json
{
  "success": true,
  "message": "保存成功"
}
```

**错误响应**:
```json
{
  "success": false,
  "message": "错误描述"
}
```

## 部署说明

### 本地开发

#### 1. 环境要求
- Python 3.7+
- 现代浏览器 (Chrome, Firefox, Safari, Edge)

#### 2. 启动服务器
```bash
cd blog
python server.py
```

#### 3. 访问地址
- 博客首页: http://localhost:8000/
- 管理后台: http://localhost:8000/admin.html

### 生产部署

#### 使用 GitHub Pages

1. 将 `blog/` 目录内容推送到 GitHub 仓库
2. 启用 GitHub Pages
3. 使用 GitHub Actions 或手动管理数据更新

#### 使用其他静态托管

1. Vercel
2. Netlify
3. Cloudflare Pages

**注意**: 生产部署需要使用其他方式管理数据更新

### 目录结构
```
blog/
├── index.html          # 博客首页
├── admin.html          # 管理后台
├── server.py           # 后端服务器
├── css/
│   ├── style.css       # 博客样式
│   └── admin.css       # 管理样式
├── js/
│   ├── app.js          # 博客逻辑
│   └── admin.js        # 管理逻辑
└── data/
    └── posts.json      # 文章数据
```

## 故障排查

### 常见问题

1. **服务器无法启动**
   - 检查端口8000是否被占用
   - 确认Python版本

2. **数据保存失败**
   - 检查文件权限
   - 确认数据格式正确

3. **页面加载异常**
   - 清除浏览器缓存
   - 检查控制台错误
   - 确认文件路径正确
