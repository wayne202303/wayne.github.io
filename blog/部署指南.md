## 本地开发

### 前置要求

- 现代浏览器（Chrome、Firefox、Safari、Edge）
- 本地服务器（可选，但推荐）

### 快速开始

#### 方式1：直接打开HTML文件

1. 下载或克隆项目到本地
2. 双击 `index.html` 文件
3. 在浏览器中查看博客

#### 方式2：使用本地服务器（推荐）

**使用 Python：**

```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```

然后访问：`http://localhost:8000`

**使用 Node.js (http-server)：**

```bash
# 安装
npm install -g http-server

# 运行
http-server -p 8000
```

然后访问：`http://localhost:8000`

**使用 VS Code Live Server：**

1. 安装 "Live Server" 扩展
2. 右键点击 `index.html`
3. 选择 "Open with Live Server"

## 部署到服务器

### 静态网站托管

#### GitHub Pages

1. 将项目推送到 GitHub 仓库
2. 进入仓库设置
3. 找到 "Pages" 部分
4. 选择分支（通常是 main）
5. 点击 "Save"

访问地址：`https://your-username.github.io/repository-name/`

#### Netlify

1. 访问 https://www.netlify.com/
2. 拖拽项目文件夹到上传区域
3. 等待部署完成

#### Vercel

1. 访问 https://vercel.com/
2. 导入 GitHub 仓库
3. 点击部署

#### 传统服务器（Apache/Nginx）

**上传文件：**

使用 FTP/SFTP 或 Git 将文件上传到服务器。

**Apache 配置示例：**

```apache
<VirtualHost *:80>
    ServerName yourdomain.com
    DocumentRoot /var/www/blog
    
    <Directory /var/www/blog>
        Options Indexes FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
```

**Nginx 配置示例：**

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/blog;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}
```

## 数据管理

### 添加新文章

1. 打开 `data/posts.json` 文件
2. 在数组末尾添加新的文章对象
3. 保存文件

**文章格式示例：**

```json
{
    "id": 13,
    "title": "你的文章标题",
    "slug": "your-article-slug",
    "date": "2024-04-16T10:00:00",
    "category": "分类名称",
    "tags": ["标签1", "标签2", "标签3"],
    "excerpt": "文章摘要，显示在列表中",
    "content": "# 文章标题\n\n文章内容，支持 Markdown 格式"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 文章唯一标识，必须唯一 |
| title | string | 是 | 文章标题 |
| slug | string | 是 | URL友好的文章标识 |
| date | string | 是 | 发布时间，ISO 8601 格式 |
| category | string | 是 | 文章分类 |
| tags | array | 是 | 文章标签数组 |
| excerpt | string | 是 | 文章摘要 |
| content | string | 是 | 文章内容，支持 Markdown |

### Markdown 支持

文章内容支持以下 Markdown 语法：

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体文本**
*斜体文本*

- 列表项1
- 列表项2

`行内代码`

\`\`\`
代码块
\`\`\`

> 引用文本

[链接文字](https://example.com)
```

## SEO优化

### 已实现的SEO功能

1. **Meta 标签**
   - 页面标题
   - 描述标签
   - 关键词标签
   - 作者标签

2. **语义化 HTML**
   - 使用 `<header>`、`<main>`、`<footer>` 等语义标签
   - 正确的标题层级（h1-h3）

3. **URL 结构**
   - 基于 slug 的友好 URL
   - 锚点导航

### 进一步优化建议

1. **添加 sitemap.xml**

创建 `sitemap.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
    <url>
        <loc>https://yourdomain.com/</loc>
        <lastmod>2024-04-15</lastmod>
        <changefreq>weekly</changefreq>
        <priority>1.0</priority>
    </url>
</urlset>
```

2. **添加 robots.txt**

创建 `robots.txt`：

```
User-agent: *
Allow: /
Sitemap: https://yourdomain.com/sitemap.xml
```

3. **Open Graph 标签**

在 `<head>` 中添加：

```html
<meta property="og:title" content="个人博客">
<meta property="og:description" content="分享技术与生活">
<meta property="og:type" content="website">
<meta property="og:url" content="https://yourdomain.com">
```

## 性能优化

### 已实现的性能优化

1. **CSS 优化**
   - 使用 CSS 变量
   - 媒体查询实现响应式
   - 避免过度动画

2. **JavaScript 优化**
   - 事件委托
   - 防抖/节流（搜索功能）
   - 延迟加载

3. **资源优化**
   - 使用 CDN 加载 Marked 库
   - 最小化 HTTP 请求

### 进一步优化建议

1. **压缩资源**

```bash
# 压缩 CSS
npm install -g clean-css-cli
cleancss -o css/style.min.css css/style.css

# 压缩 JavaScript
npm install -g terser
terser js/app.js -o js/app.min.js
```

2. **图片优化**
   - 使用 WebP 格式
   - 实现懒加载
   - 响应式图片

3. **缓存策略**

在服务器配置中添加缓存头：

```nginx
location ~* \.(css|js|jpg|jpeg|png|gif|ico|svg)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

## 常见问题

### Q: 如何修改博客标题？

A: 编辑 `index.html` 中的 `<h1 class="blog-title">` 元素。

### Q: 如何更改配色方案？

A: 编辑 `css/style.css` 中的 CSS 变量（`:root` 部分）。

### Q: 文章不显示怎么办？

A: 检查 `data/posts.json` 的 JSON 格式是否正确，使用 JSON 验证工具检查。

### Q: 如何添加新页面？

A: 创建新的 HTML 文件，更新导航栏链接，在 JavaScript 中添加相应的路由处理。
