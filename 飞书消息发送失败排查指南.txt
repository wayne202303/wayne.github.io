飞书消息发送失败排查指南
=========================

如果授权后仍然发送失败，可能的原因及解决方案：

1. 权限范围问题
-------------
- 确认是否同时需要其他相关权限
- 检查授权时是否勾选了所有需要的权限
- 可能需要的权限：im:message, im:message.send_as_user 等

2. 用户 ID 问题
--------------
- 确认 user_id 是否正确（ou_ 开头）
- 确认该用户是否在你的通讯录中
- 尝试使用 chat-id 而不是 user-id

3. 网络和代理问题
---------------
- 检查网络连接是否正常
- 检查是否需要配置代理
- 尝试访问飞书开放平台确认网络通畅

4. 应用配置问题
-------------
- 确认应用在飞书开放平台的状态是否正常
- 检查应用是否被禁用或权限被撤销
- 确认应用的可用性范围（企业内/全网）

5. 认证令牌问题
--------------
- 确认令牌是否过期（之前显示到 20:00:51）
- 尝试重新登录获取新令牌
- 检查 refresh token 是否还有效

6. 消息内容问题
-------------
- 检查消息内容是否包含敏感词
- 确认消息长度是否超出限制
- 尝试发送更简单的消息内容

7. API 限流问题
-------------
- 检查是否触发了 API 频率限制
- 等待一段时间后再尝试
- 查看错误信息中是否有 rate limit 相关提示

8. 企业策略限制
--------------
- 检查企业是否有相关安全策略限制
- 确认是否允许发送消息给特定用户
- 咨询企业管理员确认权限

排查步骤建议：
-----------
1. 先运行 `lark-cli auth status` 确认当前认证状态
2. 尝试发送给其他用户，看是否是特定用户的问题
3. 尝试使用 bot 身份发送（去掉 --as user 参数）
4. 查看完整的错误信息，可能包含更详细的原因
5. 尝试使用 chat-id 而不是 user-id 发送

获取更多帮助：
------------
- 查看完整错误信息，通常会有更具体的提示
- 访问飞书开放平台文档查看 API 说明
- 联系飞书技术支持获取帮助
